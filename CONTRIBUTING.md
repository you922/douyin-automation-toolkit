# 贡献指南 (Contributing Guide)

感谢你关注 **Douyin Automation Toolkit**！我们欢迎任何形式的贡献，包括代码改进、文档完善、Bug 报告或功能建议。

## 📝 提交 Bug 报告

如果你发现了 Bug，请创建一个 Issue 并包含以下信息：
1. **环境信息**: 操作系统版本、Python 版本、Chrome 版本。
2. **复现步骤**: 详细描述如何触发该 Bug。
3. **预期行为**: 你期望看到的结果。
4. **实际行为**: 实际发生的情况（如果有报错日志，请一并提供）。

## 💡 提交功能建议

在提交新功能前，请先搜索现有的 Issue，确保该功能尚未被提出。如果这是一个较大的改动，建议先开一个讨论帖与我们沟通实现方案。

## 🛠️ 开发流程

1. **Fork 仓库**: 点击右上角的 Fork 按钮。
2. **创建分支**: `git checkout -b feature/your-feature-name`。
3. **编写代码**: 遵循项目的代码规范，并确保逻辑清晰。
4. **测试**: 在本地运行相关命令，确保功能正常且未引入新问题。
5. **提交 PR**: 推送到你的 Fork 仓库并提交 Pull Request。

## 📏 代码规范

*   遵循 [PEP 8](https://peps.python.org/pep-0008/) Python 编码规范。
*   保持注释清晰，对于复杂的业务逻辑请提供必要的说明。
*   不要提交任何敏感信息（如 Cookie 文件、API Key、个人账号配置等）。

## ⚖️ 行为准则

请保持友好、尊重的交流氛围。我们致力于打造一个开放、包容的社区。

再次感谢你的支持！🎉
